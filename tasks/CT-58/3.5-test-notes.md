## Controller tests (`add_line_controller_test.dart`)

### `toggleHintArrows` group (3 tests)
- **defaults to off** -- Verifies `showHintArrows` is `false` after `loadData()`.
- **toggles showHintArrows on and off** -- Verifies calling `toggleHintArrows()` flips the boolean.
- **toggle preserves other state fields** -- Verifies that toggling does not corrupt `currentFen`, `pills`, or `boardOrientation`.

### `getHintArrows` group (10 tests)
- **returns empty when toggled off (default)** -- Even with root moves seeded, arrows are empty when `showHintArrows` is false.
- **returns empty when treeCache is null** -- Before `loadData()`, returns empty even if toggled on.
- **shows root move arrows at initial position** -- Seeds two root moves (e4, d4), toggles on, verifies two arrows with darker grey colour (`0x60000000`).
- **shows direct-child arrows with darker color** -- Seeds e4 with children e5 and c5, navigates to e4, verifies both arrows use darker grey.
- **shows transposition arrows with lighter color** -- Seeds two lines that transpose to the same position with different children (Nf6 direct, e5 transposition). Verifies darker and lighter colours are present.
- **deduplicates arrows: direct-child colour takes priority** -- Seeds two lines transposing to the same position with the same child SAN (Nf6). Verifies only one arrow for Nf6 and it uses the darker direct-child colour.
- **updates arrows on pill tap (position change)** -- Plays moves, taps a pill to navigate back, verifies arrows change to reflect the new position's children.
- **returns empty when no existing moves at position** -- Plays a buffered move to a position with no existing children, verifies empty arrows.
- **arrow orig and dest match the move squares** -- Seeds e4, verifies the arrow has `orig: Square.e2` and `dest: Square.e4`.
- **showHintArrows state survives other state transitions** -- Toggles on, then plays a move, flips board, takes back; verifies `showHintArrows` remains true through all transitions.
- **transposition arrows shown at buffered move position** -- Plays a followed move to a position where transposition children exist via position-key lookup, verifies arrows appear.

## Widget tests (`add_line_screen_test.dart`)

### `hint arrows toggle` group (8 tests)
- **toggle button appears in app bar when loaded** -- Verifies `Icons.visibility_off` is present after loading.
- **toggle button not shown during loading** -- Verifies no visibility icon before `pumpAndSettle`.
- **tapping toggle changes icon from off to on** -- Taps the toggle, verifies icon switches to `Icons.visibility`.
- **tapping toggle twice returns to off state** -- Double-tap cycle returns to `Icons.visibility_off`.
- **toggle tooltip shows correct text** -- Verifies "Show existing moves" when off, "Hide existing moves" when on.
- **shapes passed to ChessboardWidget when toggled on** -- Verifies `ChessboardWidget.shapes` is non-empty after toggling on with seeded root moves.
- **shapes removed from ChessboardWidget when toggled off** -- Verifies shapes are null/empty after toggling off.
- **arrows do not interfere with move entry** -- Verifies the board remains `PlayerSide.both` (interactive) while arrows are displayed.
