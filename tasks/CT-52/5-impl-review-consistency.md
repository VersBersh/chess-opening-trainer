The narrow Browser layout no longer follows the planned `boardSizeForNarrow(..., maxHeightFraction: 0.4)` path and also drops the previous `Flexible` wrapper around the board. Suggested fix: restore a height-aware narrow sizing path and keep the board in a loose `Flexible`, or apply one shared narrow height clamp across all screens if cross-screen equality must be preserved.
  2. **Major** — [add_line_screen.dart](C:/code/draftable/chess-1/src/lib/screens/add_line_screen.dart):485-529. The wide Add Line branch computes `boardSize` from the full `LayoutBuilder` constraints, then places the optional display-name banner above the board inside the same fixed-height left column. When height is the limiting dimension, that banner steals space from the board, so Add Line can render a smaller board than Drill/Browser at the same wide viewport. That breaks the wide-layout consistency requirement CT-52 was meant to enforce. Suggested fix: move dynamic content out of the area above the board in the wide branch, or reserve/subtract a fixed banner slot before calling `boardSizeForConstraints`.
