## Tests added

All tests are in the group `CT-66: Keyboard label editor layout` at the end of the existing test file.

### Test 3a: Board collapses when label editor is active and keyboard is open (narrow)
- Sets surface size to phone dimensions (390x844).
- Seeds a repertoire with `['e4', 'e5']`, starts at e5.
- Builds with `viewInsets: EdgeInsets.only(bottom: 300)` and `viewportSize: Size(390, 844)` via the static `buildTestApp` helper.
- Opens the label editor by re-tapping the focused pill.
- Asserts the board container (`ValueKey('add-line-board-container')`) has height 0.
- Asserts the label editor `TextField` is hittable (visible above keyboard).
- Asserts the `Chessboard` widget is still in the tree (clipped, not removed).

### Test 3b: Board reappears when keyboard is dismissed
- Uses `buildKeyboardTestApp` with a `ValueNotifier<EdgeInsets>` initialized to keyboard-open insets.
- Opens the label editor; verifies board is collapsed.
- Updates the notifier to `EdgeInsets.zero` (keyboard dismissed) and pumps.
- Asserts board container height is greater than 0.

### Test 3c: Board reappears when label editor is closed (keyboard still open)
- Uses `buildKeyboardTestApp` with keyboard-open insets that remain constant.
- Opens the label editor; verifies board is collapsed.
- Submits the text field (triggers `onClose`, which sets `_isLabelEditorVisible = false`).
- Asserts board container height is greater than 0, even though the keyboard is still "open".

### Test 3d: Wide layout unaffected by keyboard
- Sets surface size to 700x844 (>= 600, triggers wide layout).
- Builds with keyboard-open insets and opens the label editor.
- Asserts the Chessboard widget has height > 0 (not collapsed).

### Test 3e: Banner collapses along with board when label editor is active
- Seeds a repertoire with a labeled path (`labelsOnSan: {'e4': 'King Pawn'}`) and starts from Nf3.
- Builds with keyboard-open insets on narrow viewport.
- Opens the label editor; verifies board container has height 0.
- Asserts the banner text ("King Pawn") is not hittable (collapsed/clipped).

## Helper modifications

### `buildTestApp` (modified)
Added two optional parameters:
- `Size? viewportSize` -- when provided (or when `viewInsets` is non-zero), wraps the `AddLineScreen` in a `MediaQuery` with the given size and insets.
- `EdgeInsets viewInsets` -- defaults to `EdgeInsets.zero`.

### `buildKeyboardTestApp` (new)
For tests that need to toggle keyboard state mid-test. Takes a `ValueNotifier<EdgeInsets>` and uses `ValueListenableBuilder` so the `AddLineScreen` instance (and its controller state) is preserved across keyboard toggles. Mirrors the pattern from `drill_filter_test.dart`.

## Criteria that cannot be tested automatically

None. All acceptance criteria have corresponding widget tests. The animated transition itself (200ms AnimatedSize) is covered implicitly by `pumpAndSettle`, which waits for the animation to complete before assertions run. Visual smoothness of the animation cannot be tested in widget tests but is not an acceptance criterion.
