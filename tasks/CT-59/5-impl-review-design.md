  1. **Major — Interface Segregation / Dependency Inversion:** Adding `batchUpdateLabels` to the already broad `RepertoireRepository` in [repertoire_repository.dart](C:/code/draftable/chess-1/src/lib/repositories/repertoire_repository.dart#L47) pushes another add-line-specific transaction concern into a shared app-wide abstraction, and the ripple is visible immediately in unrelated test doubles like [drill_filter_test.dart](C:/code/draftable/chess-1/src/test/screens/drill_filter_test.dart#L189), [drill_screen_test.dart](C:/code/draftable/chess-1/src/test/screens/drill_screen_test.dart#L186), [home_screen_test.dart](C:/code/draftable/chess-1/src/test/screens/home_screen_test.dart#L141), [deletion_service_test.dart](C:/code/draftable/chess-1/src/test/services/deletion_service_test.dart#L108), and [pgn_importer_test.dart](C:/code/draftable/chess-1/src/test/services/pgn_importer_test.dart#L142). That is a concrete sign the interface is too fat: one label-only workflow change now forces unrelated consumers to know about it. `LinePersistenceService.persistLabelsOnly()` in [line_persistence_service.dart](C:/code/draftable/chess-1/src/lib/services/line_persistence_service.dart#L164) is then just a passthrough over that extra method. A better design is to depend on a narrower port for add-line persistence, or extract label writes into a focused collaborator used only by the add-line flow.
  2. **Major — Single Responsibility / Hidden Coupling / DRY:** `_persistLabelsOnly()` in [add_line_controller.dart](C:/code/draftable/chess-1/src/lib/controllers/add_line_controller.dart#L680) introduces a second bespoke reload path that manually rebuilds cache, engine, pills, transpositions, and `AddLineState`, instead of reusing a single state-rehydration abstraction like `_loadData()` in [add_line_controller.dart](C:/code/draftable/chess-1/src/lib/controllers/add_line_controller.dart#L195). It also duplicates the pending-label mapping logic already present in `_persistMoves()` at [add_line_controller.dart](C:/code/draftable/chess-1/src/lib/controllers/add_line_controller.dart#L761). The result is temporal and semantic coupling: this path only works because pill ordering, focused index semantics, and copied FEN values happen to stay aligned with `LineEntryEngine` internals. Every future state-field change now has to be updated in yet another hand-built `AddLineState(...)` block, alongside the many others already in this file such as [add_line_controller.dart](C:/code/draftable/chess-1/src/lib/controllers/add_line_controller.dart#L1028). Extract a single helper for “build label updates” and a dedicated refresh method that restores by semantic anchor (for example focused move ID / leaf move ID) rather than by copied index and FEN snapshots.
  3. **Minor — Naming / Embedded Design:** The screen now gates confirm/discard on `hasUnsavedChanges` in [add_line_screen.dart](C:/code/draftable/chess-1/src/lib/screens/add_line_screen.dart#L177), [add_line_screen.dart](C:/code/draftable/chess-1/src/lib/screens/add_line_screen.dart#L435), and [add_line_screen.dart](C:/code/draftable/chess-1/src/lib/screens/add_line_screen.dart#L914), but the discard dialog still says “You have unsaved moves” in [add_line_screen.dart](C:/code/draftable/chess-1/src/lib/screens/add_line_screen.dart#L410). After this change, that wording is no longer true for label-only edits, so the code and UI describe different concepts. Rename the dialog and related copy to “unsaved changes” so the architecture is understandable from the code and behavior alone.
  4. **Minor — File Size / SRP:** This change adds more behavior to files that are already well past the 300-line smell threshold: [add_line_controller.dart](C:/code/draftable/chess-1/src/lib/controllers/add_line_controller.dart#L1) is 1313 lines, [add_line_screen.dart](C:/code/draftable/chess-1/src/lib/screens/add_line_screen.dart#L1) is 932, [local_repertoire_repository.dart](C:/code/draftable/chess-1/src/lib/repositories/local/local_repertoire_repository.dart#L1) is 479, and the touched tests are also very large, including [add_line_controller_test.dart](C:/code/draftable/chess-1/src/test/controllers/add_line_controller_test.dart#L1) at 4271 lines and [add_line_screen_test.dart](C:/code/draftable/chess-1/src/test/screens/add_line_screen_test.dart#L1) at 3416. Extracting the label-only persistence/rebuild flow and shared fake-repository fixtures would improve locality and make future changes cheaper.
