1. **Major — Dependency Inversion / Hidden Coupling:** `RepertoireBrowserController` now accepts both raw repositories and a `DeletionService`, but nothing enforces they point to the same backing data source ([repertoire_browser_controller.dart](C:/code/misc/chess-trainer-7/src/lib/controllers/repertoire_browser_controller.dart#L74), [repertoire_browser_screen.dart](C:/code/misc/chess-trainer-7/src/lib/screens/repertoire_browser_screen.dart#L45)). The controller reads state via `_repertoireRepo/_reviewRepo` while deletes run through `_deletionService`; if these are wired differently, behavior becomes inconsistent and hard to diagnose. Suggested fix: make controller own service construction from its own repository dependencies (or accept an optional service only for tests with an invariant/assertion), so deletion and non-deletion paths cannot diverge.
